The Lab log

October 25, 1999:

We appear to have solved the security problems that forced us to upgrade the operation system a few weeks ago. The various machines have been running stable since the upgrade.

We are planning to do an unpgrade of Galton's hard disks to put webb-related data on them, but we are still waiting on the paperwork to work itself out.

August 9, 1999:

Once again we ran into disk space problems. Here are the measures taken:

1) Some accounts were deleted, their contents tared and moved to galton's /oldacc directory

2) All core dumps in everybody's home directories were deleted

To do these things,  I created two scripts. The first script "modfiles [num]" will list the login name, size of home directory and number of files modified/created in the last [num] days for each user. We killed all accounts that had not been accessed in the last four months. The second "removecores" script will use find to get paths to all files named "core" in everybody's directories and then delete them....  I guess this means its not a good idea to name a file "core" and put important stuff on it, as it will get deleted. As usual, these scripts are in /h5/morillo/admin.
