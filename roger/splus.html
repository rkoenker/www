Some notes about using Splus in Ragnar

For reasons known to all, Splus is the most used program in Ragnar and so I thought i would start a list of some of themost frequently asked questions about Splus:

  • Roger Koenker has installed a number of new libraries for use in Splus. A library for estimating log-spline models can be accessed by typing library("logspline2"). On-line help for the functions contained in this library is accessed normally with the help command, for example typing help(lsd.fit)

The functions and procedures used in Venables and Ripley's "Modern Applied Statistics with S-Plus" (1994,1997) can also be accessed with the library command. Here's a list of the available libraries:

MASS Main Library
class Classification Methods
nnet Neural networks
spatial Spatial statistics
treefix Fixes for tree pruning

For example, if you want to use the main library, you would type library("MASS")

  • If you are having problems bringing up a motif graphics window for use with Splus, you should try adding the following line to your ".cshrc" file:

LD_LIBRARY_PATH /usr/openwin/lib

  • If you need to run the Splus implementation of the quantile regression algorithm, you need to invoke Splus with the command Splus.local

  • When you are running memory intensive processes, Splus sometimes does not kill the allocated memory after you close the program, so please make sure that you check if you have left any "ghost processes" running. To check what processes are running in your account, you can use the following command

ps -aux |grep [your login name]

If any of the processes shown with this command are related to Splus, even after you have closed the program, please write me an email at morillo@ux6.cso.uiuc.edu
